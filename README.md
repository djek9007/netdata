# Netdata (docker-compose)

Этот репозиторий содержит конфигурацию `docker-compose` для запуска Netdata — инструмента мониторинга и диагностики систем в реальном времени.

**Для чего этот файл**
- **Описание:** Этот `README.md` объясняет, зачем нужен репозиторий и как быстро запустить Netdata локально.
- **Цель:** дать простые пошаговые инструкции для пользователей Windows (PowerShell) и подсказать полезные команды для отладки.

**Требования**
- **Docker:** Установлен и запущен Docker Desktop для Windows.
- **docker-compose:** В большинстве современных установок Docker Desktop `docker-compose` уже доступен.
- **Права:** Пользователь должен иметь права запускать Docker-контейнеры.

**Быстрый старт (Windows PowerShell)**
1. Откройте PowerShell с правами обычного пользователя (не обязательно администратор).
2. Перейдите в папку проекта:

```
cd d:\\projectspython\\netdata
```

3. Запустите сервисы в фоне:

```
docker-compose up -d
```

4. Откройте веб-интерфейс Netdata в браузере по адресу:

```
http://localhost:19999
```

5. Остановить и удалить запущенные контейнеры:

```
docker-compose down
```

**Полезные команды**
- **Просмотреть логи:** `docker-compose logs -f` — читать логи в реальном времени.
- **Список контейнеров:** `docker ps` — показать запущенные контейнеры.
- **Перезапустить сервис:** `docker-compose restart` — перезапустить контейнеры.

**Настройка**
- Все параметры запуска находятся в файле `docker-compose.yml`. При необходимости можно изменить тома, порты и переменные окружения в этом файле.
- После изменений перезапустите сервисы командой `docker-compose up -d`.

**Отладка и советы**
- Если Netdata не доступен на `localhost:19999`, проверьте, нет ли конфликтов портов и убедитесь, что контейнеры успешно запустились (`docker ps`).
- Проверьте логи контейнера Netdata через `docker-compose logs netdata` (замените `netdata` на имя сервиса из `docker-compose.yml`).
